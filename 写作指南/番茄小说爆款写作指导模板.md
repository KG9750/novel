# 番茄小说爆款写作指导模板

> 用途：给准备在番茄小说开书的作者做选题、开篇、连载、互动和复盘。  
> 核心目标：前三章让读者停不下，三十章内让读者愿意催更，三十万字后仍有可扩张的世界、人物关系和爽点。

## 01 核心原则

番茄小说的爆款逻辑，不是“写得漂亮就会爆”，而是“能被算法试投、能被读者读完、能让评论区说话、能稳定更新、能长期扩展”。

### 五条硬原则

| 原则 | 写作含义 | 执行标准 |
|---|---|---|
| 低门槛高概念 | 设定新，但第一章必须一眼看懂 | 10 秒内说清“主角是谁、遇到什么事、为什么必须看下去” |
| 黄金三章 | 前三章是试读样本，不是慢热铺垫 | 第一章强场面，第二章规则/困扰，第三章第一次兑现 |
| 问题接问题 | 每章都要留下下一章的阅读理由 | 章末至少有一个未解决问题 |
| 可评论性 | 读者看完想判断、站队、猜测 | 每 2-3 章设置一个选择题/预测题/立场题 |
| 稳定供给 | 更新是分发和读者习惯的基础 | 建议日更 2 章，单章 2200-3200 字 |

### 爆款公式

```text
番茄爆款潜力 = 高概念入口 x 前三章留存 x 爽点兑现速度 x 角色可传播度 x 稳定更新 x 评论互动
```

### 反向避坑

| 常见错误 | 为什么伤留存 | 改法 |
|---|---|---|
| 第一章讲设定史 | 读者还没关心世界 | 先让主角在事件里出问题 |
| 三章都在写主角惨 | 压抑没有回报 | 第三章前给一次小翻盘 |
| 主角没有主动目标 | 读者不知道该期待什么 | 每章给一个可见任务 |
| 章末没有悬念 | 读者没有翻页动力 | 用新信息、危机、选择题收尾 |
| 爽点只靠打脸 | 后期会疲劳 | 加入成长、资源、关系、世界观升级 |

## 02 选题模板

选题不是先问“我想写什么”，而是先问“读者为什么现在必须点进来”。

### 一句话卖点

填写格式：

```text
一个【普通/低位/有缺口的主角】，因为【异常能力/极端处境/身份反差】，必须在【高压目标】里完成【普通人做不到的逆袭】。
```

示例：

```text
一个随时会被裁掉的增长运营，因为能看见运动物体一秒后的轨迹，必须在被甩锅的大促事故里救回项目，并把这一秒变成改命底牌。
```

|---|---|

| 第一章能出现的强场面是什么？ |  |
| 主角最想改变的现实困境是什么？ |  |
| 读者最期待的爽点是什么？ |  |
| 这个设定能扩展到 30 万字后吗？ |  |

### 题材混搭表

| 主赛道 | 可混搭元素 | 爽点承诺 |
|---|---|---|
| 都市职业 | 微异能、商战、职场反击、创业 | 从被压榨到掌握资源 |
| 都市异能 | 机构、怪诞、职业任务、成长体系 | 现实身份和超常任务双线升级 |
| 悬疑无限 | 规则、解谜、群像、反转 | 猜规则、破局、活下去 |
| 末世天灾 | 囤货、基地、团队、资源管理 | 生存安全感和资源掌控 |
| 历史科举 | 低身份、考试、权谋、家族 | 阶层跃迁和智斗翻盘 |
| 校园轻甜 | 失恋、成长、关系反差、陪伴 | 情绪修复和关系推进 |

## 03 黄金三章模板

前三章是试读样本，目标不是“完整介绍世界”，而是让读者形成三个判断：

1. 这本书的设定我看懂了。
2. 这个主角我愿意跟。
3. 下一章我必须看。

### 第一章：场面、压力、异常

第一章必须完成：

| 功能 | 写法 |
|---|---|
| 强场面 | 直接把主角扔进危机、羞辱、事故、异常或重大选择 |
| 主角低位 | 用行动和处境展示，不用长篇履历 |
| 核心异常 | 让设定发生在主角身上，而不是旁白解释 |
| 章末钩子 | 让读者问：这到底是什么？主角怎么办？ |

第一章推荐结构：

```text
开头 300 字：主角正在出事
中段 1000 字：压力升级，主角被迫应对
后段 800 字：核心设定第一次显形
结尾 200 字：留下无法忽视的问题
```

第一章禁区：

| 禁区 | 替代方案 |
|---|---|
| 先写世界观百科 | 让世界观在事故里露一角 |
| 先写童年回忆 | 把创伤留到后面用事件触发 |
| 先写主角普通日常 | 从普通日常被打破的一刻写 |
| 先解释能力原理 | 先写能力后果 |

### 第二章：困扰、规则、现实压力

第二章重点不是马上开挂，而是写“得到设定后，主角付出什么代价”。

| 功能 | 写法 |
|---|---|
| 规则边界 | 主角试错，发现能力/处境能做什么、不能做什么 |
| 副作用 | 让金手指先带来麻烦，增加真实感 |
| 现实压力 | 工作、家人、债务、追杀、考试、规则惩罚继续压迫 |
| 章末钩子 | 主角发现这个能力也许能解决眼前问题 |

### 第三章：第一次兑现

第三章必须给读者一次明确回报。

| 功能 | 写法 |
|---|---|
| 小目标 | 不是拯救世界，而是救一个项目、赢一次考试、躲一次杀局 |
| 小翻盘 | 主角用新能力/新认知做到别人做不到的事 |
| 他人反应 | 让配角震惊、怀疑、站队或忌惮 |
| 更大问题 | 这次赢了，但引出更大的压力或秘密 |

第三章结尾句式：

```text
他终于明白，这不是病。
这是第一张底牌。
可他还不知道，已经有人注意到了他。
```

## 04 单章公式

番茄连载最怕“这一章好像没发生什么”。每章都要有一个小任务、一层阻碍、一次信息变化。

### 通用单章结构

| 段落 | 功能 | 写法 |
|---|---|---|
| 开场 | 冲突画面 | 不寒暄，直接进入事件 |
| 前半 | 当前目标 | 主角这一章具体要做什么 |
| 中段 | 阻碍升级 | 对手、规则、误会、时间限制、资源不足 |
| 后半 | 兑现或反转 | 小爽点、新线索、新关系变化 |
| 结尾 | 章末问题 | 下一章必须解决的新问题 |

### 单章填写模板

```text
本章目标：
主角为什么必须现在做：
阻碍是什么：
本章爽点是什么：
本章新增信息是什么：
章末问题是什么：
读者评论点是什么：
```

### 章末钩子类型

| 类型 | 适用场景 | 示例句式 |
|---|---|---|
| 新危机 | 动作、职场、末世 | 他刚松口气，屏幕上的第二条曲线也塌了。 |
| 新身份 | 都市、玄幻、历史 | 门外站着的人，竟然叫出了他从未告诉别人的旧名。 |
| 新规则 | 悬疑、无限、异能 | 这条规则真正可怕的地方，不是会杀人，而是只杀说真话的人。 |
| 新关系 | 言情、职场、群像 | 她没有替他解释，只把自己的名字也签在了那份方案上。 |
| 新选择 | 商战、权谋、成长 | 现在摆在他面前的不是辞职和留下，而是要不要自己开局。 |

## 05 爽点密度表

爽点不是一直打脸，而是持续给读者“期待被满足，又产生新期待”的循环。

### 密度建议

| 篇幅节点 | 读者需要什么 | 写作任务 |
|---|---|---|
| 每章 | 一个小问题和小变化 | 不写空转章 |
| 每 3-5 章 | 一个阶段性回报 | 小赢、小反转、小关系进展 |
| 每 10 章 | 一个可复述名场面 | 救人、打脸、破局、表白未遂、反杀 |
| 每 30 章 | 一个卷内转折 | 身份变化、地图升级、敌人升级、资源升级 |
| 每 80-120 章 | 一个大阶段闭环 | 第一桶金、第一次出圈、第一次败北、第一次掌局 |

### 爽点类型库

| 类型 | 读者爽在哪里 | 使用提醒 |
|---|---|---|
| 反击爽 | 被欺负的人终于还手 | 前面压迫要具体，反击要当场有效 |
| 识破爽 | 主角看穿别人看不见的问题 | 不能只靠旁白说聪明，要用结果证明 |
| 资源爽 | 钱、人脉、装备、权限到手 | 资源要马上改变下一步选择 |
| 身份爽 | 从边缘人变成关键人物 | 让旁人反应形成对照 |
| 成长爽 | 主角掌握方法，不再靠运气 | 写出训练、代价和复盘 |
| 情感爽 | 被理解、被选择、被并肩 | 感情要服务人物成长，不只撒糖 |

### 主角成长曲线

```text

```

### 反派/对手卡

| 项目 | 填写 |
|---|---|
| 他压迫主角的方式 |  |

| 项目 | 填写 |

## 07 长线连载模板

番茄长线连载要把“更新制度”和“故事扩展”一起设计。

### 30 / 90 / 180 天节奏

| 周期 | 写作目标 | 更新建议 | 运营动作 | 复盘重点 |
|---|---|---|---|---|
| 30 天 | 打出开篇，完成第一批读者筛选 | 每天 2 章，累计 10-12 万字 | 书名简介 A/B，观察评论关键词 | 慢、看不懂、主角弱是否高频出现 |
| 90 天 | 完成第一卷爆点，形成稳定标签 | 双更为主，大节点可加更 | 固定更新预告，做短内容切片 | 评论区是否开始猜剧情、站角色 |
| 180 天 | 冲中腰部长期盘 | 维持更新达成率 | 复盘封面、书名、简介、卷名 | 是否形成可传播名场面和清晰卖点 |

### 卷纲模板

| 卷次 | 字数区间 | 核心目标 | 阶段敌人 | 阶段爽点 | 卷尾钩子 |
|---|---:|---|---|---|---|
| 卷一 | 0-10 万 | 立设定，立主角，第一次翻身 |  |  |  |
| 卷二 | 10-30 万 | 扩大能力应用，拿到资源 |  |  |  |
| 卷三 | 30-60 万 | 敌人升级，主角进入更大局 |  |  |  |
| 卷四 | 60-90 万 | 翻车和认知升级 |  |  |  |
| 卷五 | 90 万+ | 建立自己的规则和势力 |  |  |  |

### 更新制度

| 项目 | 建议 |
|---|---|
| 单章字数 | 2200-3200 字 |
| 日更节奏 | 每天 2 章，重要节点可 3 更 |
| 存稿 | 发书前 20-30 章 |
| 请假 | 提前说明，不让读者突然断预期 |
| 复盘频率 | 每 10 章小复盘，每 30 章卷内复盘 |

平台规则会变化，推荐字数、福利门槛、全勤要求以作者后台最新显示为准。

## 08 互动运营模板

互动不是“求评论”，而是把读者拉进判断。

### 章末问题

| 类型 | 示例 |
|---|---|
| 选择题 | 如果是你，这时候会救人还是追线索？ |
| 预测题 | 你们觉得真正偷方案的人是谁？ |
| 立场题 | 主角这次该忍，还是直接掀桌？ |
| 关系题 | 女主这次是在试探，还是已经开始站队？ |
| 规则题 | 这条规则真的成立吗，还是有人故意误导？ |

### 站外短内容脚本

```text
一句话设定：
具体冲突画面：
主角做出的反常选择：
最后抛给观众的问题：
引导语：【小说：书名】
```

示例：

```text
他只是个被甩锅的运营，却能看见所有动作一秒后的落点。
地铁急刹那一瞬间，他看见整座站台拖出淡色轨迹线。
所有人都没反应过来，只有他提前半步伸手，拽住了即将摔下去的女同事。
如果你也能比世界快一秒，你最先会用它救人，还是改命？
【小说：一秒先机】
```

### 书名简介 A/B 表

| 测试项 | A 版 | B 版 | 观察指标 |
|---|---|---|---|
| 书名 | 短标题，强调记忆点 | 长标题，直接打卖点 | 点击率、评论第一反应 |
| 简介首句 | 世界异常 | 主角困境 | 读者是否一眼看懂 |
| 简介中段 | 设定扩展 | 人物关系 | 评论更爱猜设定还是嗑关系 |
| 简介结尾 | 大悬念 | 高情绪 | 收藏、追更、转发意愿 |

## 09 自检清单

### 开书前

| 检查项 | 通过标准 | 是否通过 |
|---|---|---|
| 一句话卖点清楚 | 陌生读者 10 秒能复述 |  |
| 前 10 章有纲 | 每章目标和钩子明确 |  |
| 角色关系有冲突 | 不是全员工具人 |  |
| 至少 5 个名场面 | 能做短内容切片 |  |
| 有 20-30 章存稿计划 | 不靠当天灵感硬撑 |  |

### 前三章

| 检查项 | 通过标准 | 是否通过 |
|---|---|---|
| 第一章有强场面 | 读者能复述画面 |  |
| 第二章有规则和代价 | 设定不是无成本外挂 |  |
| 第三章有第一次兑现 | 主角赢一次或证明一次价值 |  |
| 没有长篇设定说明 | 设定通过事件展示 |  |
| 章末都有问题 | 每章结尾能自然翻页 |  |

### 前 30 章

| 检查项 | 通过标准 | 是否通过 |
|---|---|---|
| 至少 3 次明确爽点 | 救人、打脸、赚钱、破局等 |  |
| 至少 1 次关系变化 | 搭档、敌人、女主站队等 |  |
| 至少 1 次资源变化 | 钱、职位、权限、人脉等 |  |
| 主角不再原地踏步 | 身份或能力用法有变化 |  |
| 留下下一卷入口 | 更大敌人、更大地图、更大目标 |  |
