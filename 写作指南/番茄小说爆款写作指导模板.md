# 番茄小说爆款写作指导模板

> 用途：给准备在番茄小说开书的作者做选题、开篇、连载、互动和复盘。  
> 核心目标：前三章让读者停不下，三十章内让读者愿意催更，三十万字后仍有可扩张的世界、人物关系和爽点。

## 01 核心原则

番茄小说的爆款逻辑，不是“写得漂亮就会爆”，而是“能被算法试投、能被读者读完、能让评论区说话、能稳定更新、能长期扩展”。

### 五条硬原则

| 原则 | 写作含义 | 执行标准 |
|---|---|---|
| 低门槛高概念 | 设定新，但第一章必须一眼看懂 | 10 秒内说清“主角是谁、遇到什么事、为什么必须看下去” |
| 黄金三章 | 前三章是试读样本，不是慢热铺垫 | 第一章强场面，第二章规则/困扰，第三章第一次兑现 |
| 问题接问题 | 每章都要留下下一章的阅读理由 | 章末至少有一个未解决问题 |
| 可评论性 | 读者看完想判断、站队、猜测 | 每 2-3 章设置一个选择题/预测题/立场题 |
| 稳定供给 | 更新是分发和读者习惯的基础 | 建议日更 2 章，单章 2200-3200 字 |

### 爆款公式

```text
番茄爆款潜力 = 高概念入口 x 前三章留存 x 爽点兑现速度 x 角色可传播度 x 稳定更新 x 评论互动
```

### 反向避坑

| 常见错误 | 为什么伤留存 | 改法 |
|---|---|---|
| 第一章讲设定史 | 读者还没关心世界 | 先让主角在事件里出问题 |
| 三章都在写主角惨 | 压抑没有回报 | 第三章前给一次小翻盘 |
| 主角没有主动目标 | 读者不知道该期待什么 | 每章给一个可见任务 |
| 章末没有悬念 | 读者没有翻页动力 | 用新信息、危机、选择题收尾 |
| 爽点只靠打脸 | 后期会疲劳 | 加入成长、资源、关系、世界观升级 |

## 02 选题模板

选题不是先问“我想写什么”，而是先问“读者为什么现在必须点进来”。

### 一句话卖点

填写格式：

```text
一个【普通/低位/有缺口的主角】，因为【异常能力/极端处境/身份反差】，必须在【高压目标】里完成【普通人做不到的逆袭】。
```

示例：

```text
一个随时会被裁掉的增长运营，因为能看见运动物体一秒后的轨迹，必须在被甩锅的大促事故里救回项目，并把这一秒变成改命底牌。
```

## 03 黄金三章模板

前三章是试读样本，目标不是“完整介绍世界”，而是让读者形成三个判断：

1. 这本书的设定我看懂了。
2. 这个主角我愿意跟。
3. 下一章我必须看。

### 第一章：场面、压力、异常

第一章必须完成：

| 功能 | 写法 |
|---|---|
| 强场面 | 直接把主角扔进危机、羞辱、事故、异常或重大选择 |
| 主角低位 | 用行动和处境展示，不用长篇履历 |
| 核心异常 | 让设定发生在主角身上，而不是旁白解释 |
| 章末钩子 | 让读者问：这到底是什么？主角怎么办？ |

第一章推荐结构：

```text
开头 300 字：主角正在出事
中段 1000 字：压力升级，主角被迫应对
后段 800 字：核心设定第一次显形
结尾 200 字：留下无法忽视的问题
```

第一章禁区：

| 禁区 | 替代方案 |
|---|---|
| 先写世界观百科 | 让世界观在事故里露一角 |
| 先写童年回忆 | 把创伤留到后面用事件触发 |
| 先写主角普通日常 | 从普通日常被打破的一刻写 |
| 先解释能力原理 | 先写能力后果 |

### 第二章：困扰、规则、现实压力

第二章重点不是马上开挂，而是写“得到设定后，主角付出什么代价”。

| 功能 | 写法 |
|---|---|
| 规则边界 | 主角试错，发现能力/处境能做什么、不能做什么 |
| 副作用 | 让金手指先带来麻烦，增加真实感 |
| 现实压力 | 工作、家人、债务、追杀、考试、规则惩罚继续压迫 |
| 章末钩子 | 主角发现这个能力也许能解决眼前问题 |

### 第三章：第一次兑现

第三章必须给读者一次明确回报。

| 功能 | 写法 |
|---|---|
| 小目标 | 不是拯救世界，而是救一个项目、赢一次考试、躲一次杀局 |
| 小翻盘 | 主角用新能力/新认知做到别人做不到的事 |
| 他人反应 | 让配角震惊、怀疑、站队或忌惮 |
| 更大问题 | 这次赢了，但引出更大的压力或秘密 |

第三章结尾句式：

```text
他终于明白，这不是病。
这是第一张底牌。
可他还不知道，已经有人注意到了他。
```

## 04 单章公式

番茄连载最怕“这一章好像没发生什么”。每章都要有一个小任务、一层阻碍、一次信息变化。

### 通用单章结构

| 段落 | 功能 | 写法 |
|---|---|---|
| 开场 | 冲突画面 | 不寒暄，直接进入事件 |
| 前半 | 当前目标 | 主角这一章具体要做什么 |
| 中段 | 阻碍升级 | 对手、规则、误会、时间限制、资源不足 |
| 后半 | 兑现或反转 | 小爽点、新线索、新关系变化 |
| 结尾 | 章末问题 | 下一章必须解决的新问题 |

### 单章填写模板

```text
本章目标：
主角为什么必须现在做：
阻碍是什么：
本章爽点是什么：
本章新增信息是什么：
章末问题是什么：
读者评论点是什么：
```

### 章末钩子类型

| 类型 | 适用场景 | 示例句式 |
|---|---|---|
| 新危机 | 动作、职场、末世 | 他刚松口气，屏幕上的第二条曲线也塌了。 |
| 新身份 | 都市、玄幻、历史 | 门外站着的人，竟然叫出了他从未告诉别人的旧名。 |
| 新规则 | 悬疑、无限、异能 | 这条规则真正可怕的地方，不是会杀人，而是只杀说真话的人。 |
| 新关系 | 言情、职场、群像 | 她没有替他解释，只把自己的名字也签在了那份方案上。 |
| 新选择 | 商战、权谋、成长 | 现在摆在他面前的不是辞职和留下，而是要不要自己开局。 |

## 05 爽点密度表

爽点不是一直打脸，而是持续给读者“期待被满足，又产生新期待”的循环。

### 密度建议

| 篇幅节点 | 读者需要什么 | 写作任务 |
|---|---|---|
| 每章 | 一个小问题和小变化 | 不写空转章 |
| 每 3-5 章 | 一个阶段性回报 | 小赢、小反转、小关系进展 |
| 每 10 章 | 一个可复述名场面 | 救人、打脸、破局、表白未遂、反杀 |
| 每 30 章 | 一个卷内转折 | 身份变化、地图升级、敌人升级、资源升级 |
| 每 80-120 章 | 一个大阶段闭环 | 第一桶金、第一次出圈、第一次败北、第一次掌局 |

### 爽点类型库

| 类型 | 读者爽在哪里 | 使用提醒 |
|---|---|---|
| 反击爽 | 被欺负的人终于还手 | 前面压迫要具体，反击要当场有效 |
| 识破爽 | 主角看穿别人看不见的问题 | 不能只靠旁白说聪明，要用结果证明 |
| 资源爽 | 钱、人脉、装备、权限到手 | 资源要马上改变下一步选择 |
| 身份爽 | 从边缘人变成关键人物 | 让旁人反应形成对照 |
| 成长爽 | 主角掌握方法，不再靠运气 | 写出训练、代价和复盘 |
| 情感爽 | 被理解、被选择、被并肩 | 感情要服务人物成长，不只撒糖 |

## 07 长线连载模板

番茄长线连载要把“更新制度”和“故事扩展”一起设计。

### 30 / 90 / 180 天节奏

| 周期 | 写作目标 | 更新建议 | 运营动作 | 复盘重点 |
|---|---|---|---|---|
| 30 天 | 打出开篇，完成第一批读者筛选 | 每天 2 章，累计 10-12 万字 | 书名简介 A/B，观察评论关键词 | 慢、看不懂、主角弱是否高频出现 |
| 90 天 | 完成第一卷爆点，形成稳定标签 | 双更为主，大节点可加更 | 固定更新预告，做短内容切片 | 评论区是否开始猜剧情、站角色 |
| 180 天 | 冲中腰部长期盘 | 维持更新达成率 | 复盘封面、书名、简介、卷名 | 是否形成可传播名场面和清晰卖点 |

| 卷次 | 字数区间 | 核心目标 | 阶段敌人 | 阶段爽点 | 卷尾钩子 |
|---|---:|---|---|---|---|
| 卷一 | 0-10 万 | 立设定，立主角，第一次翻身 |  |  |  |
| 卷二 | 10-30 万 | 扩大能力应用，拿到资源 |  |  |  |
| 卷三 | 30-60 万 | 敌人升级，主角进入更大局 |  |  |  |
| 卷四 | 60-90 万 | 翻车和认知升级 |  |  |  |
| 卷五 | 90 万+ | 建立自己的规则和势力 |  |  |  |

### 更新制度

| 项目 | 建议 |
|---|---|
| 单章字数 | 2200-3200 字 |
| 日更节奏 | 每天 2 章，重要节点可 3 更 |
| 存稿 | 发书前 20-30 章 |
| 请假 | 提前说明，不让读者突然断预期 |
| 复盘频率 | 每 10 章小复盘，每 30 章卷内复盘 |

平台规则会变化，推荐字数、福利门槛、全勤要求以作者后台最新显示为准。

### 番茄官方节点版连载路线

以下节点来自番茄作者课堂公开教程，适合做开书前的运营倒排。具体入口、资格和金额可能随平台调整，以作者后台、站内信和最新官方公告为准。

| 节点 | 触发条件/动作 | 写作重点 | 运营重点 |
|---|---|---|---|
| 入驻前 | 注册账号，完善信息；实名认证后才具备签约、收益和更充分曝光条件 | 准备书名、简介、前 2 万字正文 | 先把外包装和基础格式做到可审核 |
| 2 万字 | 获得第一次签约申请机会 | 前 300 字尽快展开故事；2 万字内主角、主线、冲突要清楚 | 申请前统一检查安全、原创、分段、标点、简介 |
| 5 万字 | 若前次评估失败，修改质量后到新字数段再申请 | 强化节奏、人物设定、开篇代入感和主角推动力 | 不要在没有实质修改时浪费申请机会 |
| 8 万字 | 第三次签约机会；签约作品可满足条件后开启推荐 | 第一阶段爽点应已兑现，读者知道这本书看什么 | 连续更新、每日有效过审字数达标后手动关注“开始推荐”入口 |
| 10 万字 | 全勤/星火等福利常见关键门槛；小流量池后继续扩大推荐的重要基础 | 不要在 8-10 万字断档，保持主线推进 | 避免月底极限打卡；审核未过字数不计入有效更新 |
| 20-50 万字 | 满足条件可进行一次多书名实验 | 作品卖点已经足够明确，才能测出有效标题 | 准备差异化但不误导的书名和封面组合 |
| 100 万字以上 | 满足条件可再使用一次多书名实验机会 | 长线卖点、读者群和完结预期要稳定 | 若 20-50 万字已做过实验，注意间隔和后台限制 |
| 完结前 | 故事完整、不烂尾；番外写完后再申请完结 | 收束主线、兑现承诺、补齐人物结局 | 申请完结不可撤销；完结后不能新增章节或修改正文 |

## 08 互动运营模板

互动不是“求评论”，而是把读者拉进判断。

### 章末问题

| 类型 | 示例 |
|---|---|
| 选择题 | 如果是你，这时候会救人还是追线索？ |
| 预测题 | 你们觉得真正偷方案的人是谁？ |
| 立场题 | 主角这次该忍，还是直接掀桌？ |
| 关系题 | 女主这次是在试探，还是已经开始站队？ |
| 规则题 | 这条规则真的成立吗，还是有人故意误导？ |

### 站外短内容脚本

```text
一句话设定：
具体冲突画面：
主角做出的反常选择：
最后抛给观众的问题：
引导语：【小说：书名】
```

示例：

```text
他只是个被甩锅的运营，却能看见所有动作一秒后的落点。
地铁急刹那一瞬间，他看见整座站台拖出淡色轨迹线。
所有人都没反应过来，只有他提前半步伸手，拽住了即将摔下去的女同事。
如果你也能比世界快一秒，你最先会用它救人，还是改命？
【小说：一秒先机】
```

### 书名简介 A/B 表

| 测试项 | A 版 | B 版 | 观察指标 |
|---|---|---|---|
| 书名 | 短标题，强调记忆点 | 长标题，直接打卖点 | 点击率、评论第一反应 |
| 简介首句 | 世界异常 | 主角困境 | 读者是否一眼看懂 |
| 简介中段 | 设定扩展 | 人物关系 | 评论更爱猜设定还是嗑关系 |
| 简介结尾 | 大悬念 | 高情绪 | 收藏、追更、转发意愿 |

### 社区内容运营

番茄官方教程强调，社区运营不是硬广，而是把作品转成读者愿意讨论、二创和安利的内容。

| 场景 | 推荐内容 | 避坑 |
|---|---|---|
| 章末/书末发帖 | 剧情讨论、角色站队、CP 讨论、完结感言、创作灵感 | 不发无意义水帖，不把帖子写成纯广告 |
| 书圈 | 组织剧情推理、角色图、同人二创、读者活动 | 遇到恶评先举报/禁言，不下场互骂 |
| 段评/章评 | 作者适度回复关键段落，强化读者参与感 | 不逐条解释剧情，不提前剧透 |
| 粉丝群 | 千粉后维护核心读者，收集反馈、预告更新 | 群管理要稳定，不把读者反馈当即时改纲命令 |
| 书荒推书 | 选“拯救书荒人数多、回帖少、与作品精准匹配”的话题 | 不刷帖，不蹭与作品无关的热帖 |

推书文案公式：

```text
符合题主要求的一句话定位 + 高能冲突/悬念 + 真实阅读感受 + 书名/书卡
```

示例：

```text
想看都市异能但不想看无脑开挂，可以试试这本。
主角只能看见下一秒的轨迹，救人、破局、翻身都只有一秒窗口，赢得很爽，但每次使用都会把自己逼进更危险的局。
最戳我的是它不是单纯打脸，而是主角一点点学会用这一秒改命。
```

## 10 番茄官方课堂补充

本节来自对番茄小说作家专区“作者课堂/适用对象：所有品类”21 篇公开教程的整理，抓取时间为 2026-04-25。官方教程中有些早期文章与 2026 年新规则存在门槛差异，涉及签约、推荐、福利、全勤、稿费结算时，优先以后台最新规则和最新官方文章为准。

### 官方教程的核心结论

| 主题 | 对写作者的实际含义 | 执行动作 |
|---|---|---|
| 签约不是只看文笔 | 评估会看题材适配、节奏、人物设定、开篇、代入感、主线、故事性和安全合规 | 2 万字前要形成“主角推动主线”的明确样本 |
| 推荐不是人工排榜 | 作品推荐由状态、持续更新、内容质量、读者行为数据共同影响 | 不要只等编辑，持续修前文、稳更新、看读完率 |
| 收益主要来自听读 | 阅读/听书广告分成是大部分作者主要收入来源，互动、渠道、版权、福利是补充 | 爆款目标要同时服务阅读留存和听书可听性 |
| 规则会影响收益 | 全勤、星火等按有效过审字数、收益门槛、作品状态、数据可信度判定 | 更新和修改要给审核留余量，避免月底极限操作 |
| 标签决定目标读者 | 标签不匹配会吸引错误读者，拉低读完率和口碑 | 开书前就把题材、主题、角色、情节标签设计进内容 |
| 书名封面可实验 | 多书名实验能测点击，但不能用无关噱头骗点击 | 书名要换卖点、换角度，但不能换题材承诺 |
| 社区能放大作品 | 书圈、段评、粉丝群、图文发帖能提高粘性和讨论度 | 设计“可讨论剧情”和“可二创角色”，不要只求夸 |
| 完结也是运营节点 | 完结申请不可撤销，完结后正文和番外无法再新增修改 | 番外、伏笔回收、结局情绪全部完成后再提交 |

### 签约前 2 万字专项标准

番茄教程把签约评估落到了很具体的开篇要求：作品要原创、安全、格式正常，并且前 2 万字要能证明它是一部有故事性的小说。

| 检查点 | 合格写法 | 常见失败 |
|---|---|---|
| 书名 | 贴合主题、背景时代和核心卖点，能勾起点击欲 | 太泛、太文艺、看不出题材 |
| 简介 | 突出剧情亮点和主角设定亮点，可用互动小剧场加分 | 只有世界观、没有主角处境 |
| 前 300 字 | 出主角、出背景、出事件，不拖铺垫 | 大段设定史、氛围散文、无人物行动 |
| 主角 | 对主线有明确掌控力和推动作用 | 主角只是旁观者，剧情靠别人推进 |
| 主线 | 读者知道主角接下来要解决什么问题 | 情节流水账、没有目标、没有阻碍 |
| 节奏 | 情节环环相扣，章末有钩子 | 一章一件小事但没有变化 |
| 格式 | 分段、标点、章节卡点适合手机阅读 | 全文不分段、标点混乱、段落过长 |

### 推荐与数据复盘

番茄教程反复强调，推荐是读者行为驱动的扩大过程：系统先把作品展示给一批读者，再根据阅读行为继续圈定同类读者。

| 数据/现象 | 说明 | 修正方向 |
|---|---|---|
| 点击低 | 书名、封面、简介、标签没有让目标读者产生预期 | 重写一句话卖点；强化题材关键词和主角困境 |
| 读完率低 | 内容承诺和正文不一致，或开篇慢、主角弱、爽点迟 | 重修前三章；提前冲突和第一次兑现 |
| 评论说看不懂 | 信息投放太密或设定解释太抽象 | 用事件展示规则，减少百科式说明 |
| 追读弱 | 单章没有目标和章末问题 | 每章补一个任务、阻碍、变化、钩子 |
| 流量波动 | 个性化推荐会随数据实时调整，波动正常 | 不追问“何时上榜”，按章节表现持续优化 |

### 标签与多书名实验

标签和多书名实验都服务同一个目标：让正确读者点进来，并且留下来。

| 项目 | 官方要点 | 写作转化 |
|---|---|---|
| 主分类 | 必选且只能一个，是最核心类别 | 不要用边缘元素冒充主赛道 |
| 主题/角色/情节 | 可选，最多各两个，用来细分卖点 | 标签要在前 3-10 章真实出现 |
| 签约后标签 | 签约后作者不能自行修改，审核会按内容调整 | 开书前先确定作品到底承诺什么 |
| 多书名数量 | 除原书名外，单次实验最多 5 组 | 准备 3-5 个不同角度的标题 |
| 多书名周期 | 通常第 7-10 天产出最优书名 | 实验期不要频繁干扰正文方向 |
| 结果确认 | 结果产出后 72 小时内选择，否则可能自动确认 | 关注站内消息，避免错过操作 |
| 风险 | 书名骗点击会拉低留存和整体数据 | 可以吸睛，不能偏离人设、题材、冲突 |

### 更新、审核与全勤避坑

| 风险 | 后果 | 预防 |
|---|---|---|
| 月底最后一天极限打卡 | 审核未计入有效字数，可能错过全勤或推迟资格 | 至少提前 1-2 天完成关键字数 |
| 次月 1 日修改章节 | 修改后重新审核不过，可能影响上月打卡字数 | 当月内完成章节修改 |
| 定时章节卡在前面 | 后续章节无法越过前序章节进入审核/发布 | 先处理前序定时章节，再发布后续章节 |
| 距离发布时间不足 30 分钟改定时 | 时间锁定，无法修改 | 定时发布至少提前半小时检查 |
| 审核中章节调序 | 定时/审核中章节不能调序 | 发布成功后三天内再处理可调章节 |
| 违规、停止推荐或下架 | 不符合平台福利发放标准 | 先处理安全合规和站内信提示 |
| 刷量刷收益 | 可能扣除福利并取消后续资格 | 不参与任何异常数据操作 |

### 头条推荐标题模板

符合条件的作品可为前 50 章写今日头条推荐标题。标题不是目录章名，而是面向信息流的点击承诺。

| 标题要求 | 写法 |
|---|---|
| 15-30 字 | 一句话说清人物、处境、冲突或结果 |
| 有故事感 | 用“谁在什么困境里做了什么反常选择” |
| 可用数字 | 数字能让代价、时间、规模更清晰 |
| 有悬念和冲突 | 让读者想知道原因或后果 |
| 拒绝标题党 | 不用震惊体，不写与正文无关的噱头 |
| 不写第几章 | 不要把“第 xx 章”当推荐标题内容 |

填写模板：

```text
主角身份/处境 + 关键冲突 + 反常选择/结果
```

示例：

```text
被甩锅的运营看见下一秒轨迹，提前一步截住了致命事故
```

### 完结前检查

| 检查项 | 标准 |
|---|---|
| 主线 | 核心矛盾已解决，不靠突然旁白收尾 |
| 人物 | 主角、搭档、主要对手都有结局位置 |
| 番外 | 想写的番外已经写完并发布 |
| 伏笔 | 重要伏笔已回收，未回收项有合理留白 |
| 状态 | 确认不需要再修改正文或新增章节 |
| 预期 | 不把“完结推”当作流量保证，完结后仍按作品质量推荐 |
